# CWRU Working Paper

## Research Question

**To What Extent Does Data Augmentation Compensate for Leakage-Safe Splitting?** A factorial experiment (2 models × 2 split protocols × 10 augmentation conditions × 5 random seeds = 200 training runs) testing whether data augmentation recovers the generalization gap caused by removing adjacent-window leakage in CWRU bearing fault diagnosis.

See `ars/research_question_brief.md` for full details.

## Datasets

- **CWRU Bearing Dataset** — Drive End accelerometer, 12 kHz sampling, 40 .mat files, 4 classes (Normal, IR, OR, Ball), 3 fault diameters (0.007/0.014/0.021"), 4 motor loads (0/1/2/3 HP). Source: https://engineering.case.edu/bearingdatacenter.
- Dataset card: `data_manifest/dataset_card_cwru.md`
- Data-copy audit: `data_manifest/local_data_copy_audit.md` (the E-drive copy is damaged; the D-drive and A10 selected-file hashes match)

## Reproducing Preprocessing

```bash
# Install dependencies
pip install -r environment/requirements.txt

# To reproduce the audited A10 software stack exactly instead:
pip install -r environment/requirements-a10-lock.txt

# Configure data path
cp src/config.example.py src/config.py
# Either edit DATA_ROOT in src/config.py, or set CWRU_DATA_ROOT to the
# directory containing 01_Normal_Baseline/ and 02_12k_Drive_End_Fault/.
# The default device is CPU. Set CWRU_DEVICE=cuda for a CUDA-capable run.

# Validate all selected files and materialize the exact split assignments
python src/audit_dataset.py
```

The preprocessing pipeline:
1. Selects exactly 40 `.mat` recordings (4 Normal, 12 IR, 12 OR@6, 12 Ball), verifies every file is readable, records SHA-256 hashes, and extracts the DE accelerometer channel
2. Parses filename for fault_type, fault_diameter, motor_load
3. Segments into sliding windows: 1024 samples with 50% overlap for both split protocols
4. Z-score normalizes using training-set statistics only
5. Applies the seeded transform once per run to replace the normalized training windows; validation/test remain unaugmented, and transforms are not resampled per epoch

## Reproducing Training

```bash
# Full pipeline (data audit + 200 main runs + 30 grouping-ablation runs
# + mechanism validation + analysis + figures)
python src/run_all.py

# Training only
python src/train.py
```

Training configuration:
- Models: 1D-CNN (4 conv layers) and 2D-CNN (4 conv layers, STFT input: n_fft=128, hop=64)
- Optimizer: Adam, learning rate 0.001
- Loss: CrossEntropy
- Batch size: 128
- Epochs: 50
- Seeds: [42, 123, 456, 789, 1024]
- Split: 60/20/20 train/val/test, stratified by class

## Known Limitations

1. **Single dataset**: CWRU is an artificial-fault laboratory benchmark; results may not transfer to industrial field data.
2. **Single sensor**: Only drive-end accelerometer (12 kHz) used. Fan-end (48 kHz) results may differ.
3. **Fixed augmentation parameters**: Parameters chosen a priori; sensitivity not explored.
4. **Offline augmentation**: Each transform is sampled once per run and replaces rather than expands the training partition; online per-epoch resampling may behave differently.
5. **CNN architectures only**: Results may differ for Transformer or MLP architectures.
6. **50-epoch ceiling**: The best-validation epoch is descriptive and does not prove formal convergence.
7. **Physical metric scope**: Symmetric envelope-spectrum band-energy fidelity measures preservation, not causal physical reasoning.
8. **Benchmark scope**: The study deliberately uses the manual's CWRU-only minimal-project option; no second-dataset generalization claim is made.
9. **Grouping stress-test comparability**: Load grouping uses 2/1/1 of four groups, while fault-size grouping uses 1/1/1 of three groups. Exact assignments are released, but the accuracy ordering is descriptive because training coverage is not matched.
