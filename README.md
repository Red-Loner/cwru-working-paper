# CWRU Working Paper

## Research Question

**To What Extent Does Data Augmentation Compensate for Leakage-Safe Splitting?** A factorial experiment (2 models × 2 split protocols × 10 augmentation conditions × 5 random seeds = 200 training runs) testing whether data augmentation recovers the generalization gap caused by removing adjacent-window leakage in CWRU bearing fault diagnosis.

See `ars/research_question_brief.md` for full details.

## Datasets

- **CWRU Bearing Dataset** — Drive End accelerometer, 12 kHz sampling, 40 .mat files, 4 classes (Normal, IR, OR, Ball), 3 fault diameters (0.007/0.014/0.021"), 4 motor loads (0/1/2/3 HP). Source: https://engineering.case.edu/bearingdatacenter.
- Dataset card: `data_manifest/dataset_card_cwru.md`

## Reproducing Preprocessing

```bash
# Install dependencies
pip install -r environment/requirements.txt

# Configure data path
cp src/config.example.py src/config.py
# Edit src/config.py: set DATA_ROOT to your CWRU 12kHz Drive End data directory
```

The preprocessing pipeline:
1. Loads 40 .mat files, extracts DE accelerometer channel
2. Parses filename for fault_type, fault_diameter, motor_load
3. Segments into sliding windows: 1024 samples, 50% overlap (random split) / 0% (recording split)
4. Z-score normalizes using training-set statistics only
5. Applies augmentation to training set only

## Reproducing Training

```bash
# Full pipeline (200 training runs + mechanism validation + analysis + figures)
python src/run_all.py

# Training only
python src/train.py
```

Training configuration:
- Models: 1D-CNN (4 conv layers) and 2D-CNN (4 conv layers, STFT input: n_fft=128, hop=64)
- Optimizer: Adam, learning rate 0.001
- Loss: CrossEntropy
- Batch size: 128
- Epochs: 50
- Seeds: [42, 123, 456, 789, 1024]
- Split: 60/20/20 train/val/test, stratified by class

## Known Limitations

1. **Single dataset**: CWRU is an artificial-fault laboratory benchmark; results may not transfer to industrial field data.
2. **Single sensor**: Only drive-end accelerometer (12 kHz) used. Fan-end (48 kHz) results may differ.
3. **Fixed augmentation parameters**: Parameters chosen a priori; sensitivity not explored.
4. **CNN architectures only**: Results may differ for Transformer or MLP architectures.
5. **50-epoch ceiling**: ~6.7% of runs converged in final 10% of epochs.
6. **R_energy metric confound**: All augmentations increase broadband energy in fault bands (R_energy ≥ 1.0), confounding physical fidelity with noise injection level.
7. **5 of 200 epoch logs missing** — corresponding runs did not produce readable log files (possibly early convergence with non-standard exit codes).
